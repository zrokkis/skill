# CIDI 框架 (CIDI Framework)

CIDI 框架通过将复杂的任务分解为四个基本组件，简化了 AI 交互。它强调结构化的清晰度，旨在减少错误、提高生产力并确保输出与目标高度一致。

## 核心组成部分 (Core Components)

1. **背景 (Context)**：定义 AI 的角色并建立任务范围，为输出奠定坚实的基础。
2. **指令 (Instructions)**：提供清晰、可操作的步骤供 AI 遵循，确保输出的一致性。
3. **细节 (Details)**：指定具体要求、约束条件或补充信息，以精细化 AI 的回应。
4. **输入 (Input)**：提供相关数据、文档或背景信息，引导 AI 产生基于事实的深入理解。

---

## 提示词示例 (Prompt Examples)

### 示例 1：项目任务说明 (Project Task Definition)
* **背景 (Context)**：你是一位资深的项目经理，正在为开发团队分配任务。
* **指令 (Instructions)**：请根据提供的项目目标，撰写一份详细的任务说明书。
* **细节 (Details)**：必须包含任务名称、负责人占位符、验收标准和截止日期格式。语言要通用且专业。
* **输入 (Input)**：项目目标是“在下个季度上线新的会员积分系统”。

### 示例 2：内容创作辅助 (Content Creation Support)
* **背景 (Context)**：你是一位擅长 SEO 的内容营销专家。
* **指令 (Instructions)**：请根据输入的主题，生成 3 个吸引人的博客标题及简要大纲。
* **细节 (Details)**：标题必须包含核心关键词，大纲需采用 H2 结构，总字数控制在 200 字以内。
* **输入 (Input)**：本次博客的主题是“2026 年远程办公的最佳工具”。
