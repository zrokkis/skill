# ROSES 框架 (ROSES Framework)

ROSES 框架是一种高度结构化的提示词工程策略，旨在确保 AI 的每一次互动都能精准对齐业务目标并具备实际的可操作性。通过整合角色定义、目标设定、场景描述、预期方案阐述和步骤规划，它为解决复杂挑战提供了一个精细化的蓝图。

## 核心组成部分 (Core Components)

1. **角色 (Role)**：定义 AI 在处理提示词时的功能或视角，例如“业务分析师”或“网络安全专家”，为其参与设定舞台。
2. **目标 (Objective)**：阐明期望 AI 达成的具体目标，为它的努力提供一个清晰的靶向。
3. **场景 (Scenario)**：呈现 AI 应在其中运行的具体情况或上下文，增加任务的相关性和特异性。
4. **预期方案 (Expected Solution)**：描述 AI 应力求产出的理想结果或方案类型，引导其响应满足用户需求。
5. **步骤 (Steps)**：概述 AI 在制定响应时应遵循的操作或考虑因素的逻辑序列，确保采用结构化的方式解决问题。

## 提示词示例 (Prompt Examples)

### 示例 1：订阅服务流失率优化 (Customer Retention Strategy)
* **任务**：提升一家订阅制服务的客户留存率。
* **结构化提示词**：
    * **角色 (Role)**：作为一名专注于客户参与度的业务分析师。
    * **目标 (Objective)**：制定一套策略，在未来一年内将客户留存率提高 15%。
    * **场景 (Scenario)**：尽管客户满意度评分很高，但该订阅服务的续订率近期出现了逐渐下降的趋势。
    * **预期方案 (Expected Solution)**：提出一份多维度的留存计划，涵盖改进领域、激励措施和互动战术。
    * **步骤 (Steps)**：分析当前的客户反馈、识别流失趋势，并针对行业标准进行基准测试。

### 示例 2：校园健康公平推广 (School Wellness Fair)
* **任务**：为学校健康展会制作宣传页。
* **结构化提示词**：
    * **角色 (Role)**：你是一位具有儿童肥胖预防经验的儿科营养专家。
    * **目标 (Objective)**：制作一份关于“如何打包健康午餐”的教育传单。
    * **场景 (Scenario)**：目标受众是参加学校健康展会、时间紧迫的家长。
    * **预期方案 (Expected Solution)**：一份包含实用技巧、对比表格和结论的图文并茂的传单草案。
    * **步骤 (Steps)**：先提供三个简单快捷的配餐公式，然后解释每种搭配的营养价值，最后给出一个具体的购物清单建议。
