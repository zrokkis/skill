# 利弊分析框架 (Pros and Cons Analysis Framework)

利弊分析（优缺点分析）框架通过全面权衡决策的正面与负面因素，帮助用户进行理性的决策评估。它能够消除主观偏见，确保在多个选项中找到最佳的平衡点。

## 核心步骤 (Core Steps)

1. **识别选项 (Identify Options)**：列出所有正在考虑的潜在方案或解决方案。
2. **列出优点 (List Pros)**：针对每个选项，概述其优势或益处。
3. **列出缺点 (List Cons)**：同样地，详细写出其劣势或潜在的风险。
4. **评估与对比 (Evaluate and Compare)**：综合权衡各选项的利弊，确定哪个方案在利益与挑战之间达到了最佳平衡，从而做出明智的决策。

## 提示词示例 (Prompt Examples)

### 示例 1：职业选择评估 (Career Path Evaluation)
* **任务**：对比“在初创公司工作”与“在大公司工作”。
* **结构化提示词**：
    * **选项 (Options)**：初创公司 vs 大公司。
    * **优点 (Pros)**：
        * 初创公司：灵活度高、成长空间巨大、权责更广泛。
        * 大公司：稳定、福利好、体系完善、品牌背书。
    * **缺点 (Cons)**：
        * 初创公司：风险高、加班严重、资源匮乏。
        * 大公司：螺丝钉文化、晋升慢、流程官僚。
    * **评估 (Evaluate)**：作为一名追求快速成长且能承受高压的应届生，哪种环境更适合我当前的发展阶段？

### 示例 2：技术架构选择 (Technical Architecture Choice)
* **任务**：决定是否使用单体架构或微服务。
* **结构化提示词**：
    * **选项 (Options)**：单体架构 vs 微服务。
    * **优点 (Pros)**：列出两者的开发效率和部署便利度。
    * **缺点 (Cons)**：评估两者的扩展性、维护成本和系统复杂度。
    * **评估 (Evaluate)**：对于一个只有 5 人的初创开发团队，且需要快速验证 MVP，请给出建议。
