# RACEF 框架 (RACEF Framework)

RACEF 框架是一种由五个步骤组成的结构化方法，旨在通过提高准确性、相关性和清晰度来优化 AI 提示词（Prompts）。

## 核心组成部分 (Core Components)

1. **改写 (Rephrase)**：优化查询的措辞，以获得更精确、更相关的结果。
2. **追加 (Append)**：添加细节、规格或限制条件，引导 AI 生成更有针对性的输出。
3. **情境化 (Contextualize)**：提供背景信息或具体情境，使 AI 的回应符合你的特定需求。
4. **示例 (Examples)**：包含示例输入或预期的输出结果，以明确预期并展示所需的格式。
5. **后续行动 (Follow-Up)**：通过让 AI 提出问题或利用可操作的见解来优化其回应，鼓励迭代改进。

## 提示词示例 (Prompt Examples)

### 示例 1：市场调研报告 (Market Research Report)
* **任务**：为一家科技初创公司创建市场调研报告。
* **结构化提示词**：
    * **改写 (Rephrase)**：分析可穿戴技术的最新趋势。
    * **追加 (Append)**：包含关键增长因素和消费者偏好。
    * **情境化 (Contextualize)**：重点关注北美市场，时间跨度为过去五年。
    * **示例 (Examples)**：提供该领域三个成功产品的案例。
    * **后续行动 (Follow-Up)**：提出进入该市场的可操作性建议。

### 示例 2：编程辅助 (Coding Help)
* **任务**：编写 Python 脚本以自动组织文件。
* **结构化提示词**：
    * **改写 (Rephrase)**：编写一个 Python 脚本，根据文件扩展名对文件夹中的文件进行分类。
    * **追加 (Append)**：使用 `os` 和 `shutil` 库。
    * **情境化 (Contextualize)**：脚本应将文件整理到名为“Images”、“Documents”和“Videos”的文件夹中。
    * **示例 (Examples)**：例如，`.jpg` 应放入“Images”，`.pdf` 应放入“Documents”。
    * **后续行动 (Follow-Up)**：解释如何安排该脚本每日定期运行。

### 示例 3：内容创作 (Content Creation)
* **任务**：起草一篇关于可持续生活的博客文章。
* **结构化提示词**：
    * **改写 (Rephrase)**：写一篇关于零浪费生活方式好处的博客文章。
    * **追加 (Append)**：目标受众为年轻专业人士，语调要具有激励性。
    * **情境化 (Contextualize)**：重点关注日常生活中的简单变化，如使用重复利用袋和堆肥。
    * **示例 (Examples)**：提及推崇可持续发展的品牌或实践方法。
    * **后续行动 (Follow-Up)**：为这篇博客提供 5 个引人注目的标题。
