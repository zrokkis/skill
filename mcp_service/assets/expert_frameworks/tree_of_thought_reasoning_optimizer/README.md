# 思维树框架 (Tree of Thought Framework - ToT)

思维树框架 (Tree of Thought) 是一种非线性的问题解决方法，旨在增强 AI 模型的复杂推理能力。它将任务分解为一个个独立的“想法”或步骤，允许 AI 探索多条路径、评估潜在的解决方案，并在某条路径证明无效时进行回溯。这种方法模仿了人类的启发式搜索过程，非常适合需要战略规划、创意头脑风暴和多阶段决策的任务。

## 核心方法论 (Core Methodology)

该框架的结构类似于一棵树，由以下核心组件组成：

* **根节点 (Root)**：起点，即需要解决的核心问题或中心疑问。
* **节点 (Nodes)**：推理过程中的独立想法、步骤或要素。
* **边 (Edges)**：连接节点的决策逻辑或关系。
* **分支 (Branches)**：允许并行探索想法的不同路径或方法。
* **叶节点 (Leaves)**：最终的结果或决策。通过评估这些“叶子”来选择最优结果。

## 提示词示例 (Prompt Examples)

### 示例 1：业务增长决策 (Decision-Making for Business Growth)
* **根节点 (Root)**：作为一名业务顾问，提出扩展小型电商业务的策略。
* **节点 (Nodes)**：探索市场扩张、产品多样化和营销投入。
* **边 (Edges)**：根据收入潜力、获客成本和可扩展性评估各个选项。
* **叶节点 (Leaves)**：推荐最可行的策略并列出具体实施步骤。

### 示例 2：创意问题解决 (Creative Problem Solving)
* **根节点 (Root)**：作为一名物流经理，设计一个减少城市地区送货延迟的方案。
* **节点 (Nodes)**：考虑路径优化、增加资源投入以及替代运输方式。
* **边 (Edges)**：根据成本、速度和客户满意度对比各种方法。
* **叶节点 (Leaves)**：确定最佳方案并概述一个试点项目。
