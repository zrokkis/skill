# VIC 框架 (VIC Framework)

VIC 框架是一个精简而强大的三维提示词模型，由 **Voice (声音/角色)**、**Intent (意图/目标)** 和 **Context (情境/背景)** 组成。它强调通过设定独特的“声音”来控制与用户的交互风格，通过明确“意图”来锁定任务目标，并通过“情境”来提供必要的背景信息。这个框架非常适合需要快速构建、但又要求高质量输出的场景。

## 核心组成部分 (Core Components)

1. **声音 (Voice)**：不仅是语气，更是 AI 的“人设”或“角色”。它定义了 AI 应该“像谁一样说话”，包含了风格 (Style)、语气 (Tone) 和角色 (Persona)。
    *   *例如*：“作为一名严谨的德国工程师”、“像一位慈祥的老祖母”、“模仿莎士比亚的戏剧腔调”。
2. **意图 (Intent)**：明确任务的核心目标。告诉 AI 你到底想要它做什么，或者你希望通过这个任务达成什么效果。
    *   *例如*：“为了说服客户购买产品”、“为了解释复杂的量子力学概念”、“为了生成一段 Python 代码”。
3. **情境 (Context)**：提供完成任务所需的背景信息、限制条件和适用场景。没有情境，AI 的回答往往是泛泛而谈的。
    *   *例如*：“面对一群只有小学文化水平的受众”、“基于 2023 年的市场数据”、“假设我们在一个资源匮乏的孤岛上”。

## 提示词示例 (Prompt Examples)

### 示例 1：客户投诉处理 (Customer Complaint)
* **任务**：回复一位因送货延误而愤怒的客户。
* **结构化提示词**：
    * **声音 (Voice)**：你是一位极具同理心、专业且冷静的高级客户服务经理。语气要诚恳、歉意但有解决方案。
    * **情境 (Context)**：客户购买的生日礼物比承诺时间晚到了 3 天，导致他错过了送礼的最佳时机。客户现在非常生气，威胁要通过社交媒体曝光我们。
    * **意图 (Intent)**：平息客户的怒火，挽回公司的声誉，并提供一个不仅能补偿损失、还能让他感到惊喜的赔偿方案，最终让他撤回投诉。

### 示例 2：代码审查 (Code Review)
* **任务**：审查一段初级开发者的代码。
* **结构化提示词**：
    * **声音 (Voice)**：你是一位经验丰富、严格但乐于助人的首席架构师 (CTO)。你的评论应该一针见血，但同时具有教育意义。
    * **情境 (Context)**：这段代码是一个 Python 后端 API 接口，用于处理高并发的用户登录。目前的实现方式使用了简单的同步处理，可能存在性能瓶颈。
    * **意图 (Intent)**：指出代码中的性能隐患，解释为什么这样做在高并发下会崩溃，并给出具体的异步 (Async/Await) 重构建议，帮助开发者提升技术水平。
