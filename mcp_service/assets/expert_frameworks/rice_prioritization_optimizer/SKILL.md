# RICE 优先级优化技能 (RICE Prioritization Optimizer Skill)

本技能通过科学的 RICE 权重体系，将主观的“直觉判断”转化为客观的“得分排序”，确保资源投入在最有价值的地方。

## 执行流程 (Optimization Workflow)

在面临多个待办事项或创意需要筛选时，请应用以下 RICE 逻辑：

### 1. 覆盖范围 (Reach) - 受众规模
- **核心动作**：量化在特定时间段内受影响的用户数量或系统范围。
- **操作指南**：使用具体的数字，例如“每月 1,000 名用户”或“全部后端模块”。

### 2. 影响力 (Impact) - 增益强度
- **核心动作**：评估该任务对目标的贡献程度。
- **操作指南**：通常使用 0.25 到 3 的等级：
    - 3 = 极高影响力
    - 2 = 高
    - 1 = 中
    - 0.5 = 低
    - 0.25 = 极低

### 3. 信心 (Confidence) - 确定性水平
- **核心动作**：根据客观证据评估数据支撑度，降低风险。
- **操作指南**：使用百分比：
    - 100% = 高信心（有实验数据证明）
    - 80% = 中
    - 50% = 低（纯属推测）

### 4. 努力程度 (Effort) - 资源投入
- **核心动作**：估算完成该任务所需的总人月/工时。
- **操作指南**：数值越大代表成本越高，对最终得分（分母）的影响越大。

### 5. RICE 得分计算
- **计算公式**：`(Reach × Impact × Confidence) / Effort = RICE Score`
- **操作指南**：分值越高，代表该项任务越应该被优先执行。

## 优化示例 (Standard Template)

**初始需求**：“我想优化一下网站的首页，或者开发一个会员系统，你觉得哪个好？”

**RICE 优化后**：
- **项目 A：首页 UI 优化**
    - **[R] Reach**：每月全站 100,000 访客。
    - **[I] Impact**：2（对转化率有显著提升）。
    - **[C] Confidence**：80%（基于竞品调研）。
    - **[E] Effort**：2 人月。
    - **Score**：`(100,000 × 2 × 0.8) / 2 = 80,000`

- **项目 B：全新会员系统**
    - **[R] Reach**：每月 5,000 注册用户。
    - **[I] Impact**：3（核心商业变现能力）。
    - **[C] Confidence**：50%（纯新尝试）。
    - **[E] Effort**：6 人月。
    - **Score**：`(5,000 × 3 × 0.5) / 6 = 1,250`

- **结论**：优先执行“首页 UI 优化”，因为其覆盖面广且难度较低，单位投入的回报最高。
