# TAG 短链式行动优化技能 (TAG Concise Action Skill)

本技能通过最简化的“三点一线”模型（做什么、怎么做、做成什么样），剔除所有修饰性废话，直接驱动 AI 进入高效率的生产模态，确保交付物与目标严丝合缝。

## 执行流程 (Optimization Workflow)

在面临需要快速、准确产出成果的单一任务时，请应用以下 TAG 逻辑进行强制压缩：

### 1. 任务 (Task) - 领域锁定
- **核心动作**：一句话定义任务的本质。
- **操作指南**：使用“写一封信”、“做一个代码 Review”、“分析一下这份预算”等名词短语直接开场。

### 2. 行动 (Action) - 执行指令
- **核心动作**：指定核心算法或处理逻辑。
- **操作指南**：要求 AI 进行具体的加工动作，例如“提取关键词”、“重写为正式语气”、“检查逻辑漏洞”等。

### 3. 目标 (Goal) - 成果定义
- **核心动作**：告知 AI 最终的衡量标准。
- **操作指南**：指定产出的用途或效果，如“为了让老板一分钟看懂”、“为了消除潜在的运行时错误”。

## 优化示例 (Standard Template)

**初始需求**：“帮我处理一下这个 SQL。”

**TAG 优化后**：
- **[T] Task**：优化一个执行缓慢的 MySQL 查询语句。
- **[A] Action**：分析 EXPLAIN 命令的输出结果，识别缺失的索引并重写子查询。
- **[G] Goal**：将查询耗时降低到 100ms 以内，确保首页加载不产生卡顿。
