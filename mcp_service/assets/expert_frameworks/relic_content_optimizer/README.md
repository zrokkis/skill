# RELIC 框架 (RELIC Framework)

RELIC 框架通过将提示词分解为五个关键组件，简化了提示词的创建过程。这种结构化的方法能够确保提示词在各种应用场景下都能交付具有可操作性的结果。

## 核心组成部分 (Core Components)

1. **角色 (Role)**：指定 AI 扮演的角色，如顾问、策略师或创作者，以便量身定制回应。
2. **强调 (Emphasis)**：指出请求中最关键的方面，引导 AI 的关注焦点。
3. **限制 (Limitation)**：设定边界，确保回答简洁、相关且符合品牌调性。
4. **信息 (Information)**：提供必要的背景资料或上下文，确保输出结果的准确性。
5. **挑战 (Challenge)**：明确定义问题或任务，为 AI 创建清晰的目标。

## 提示词示例 (Prompt Examples)

### 示例 1：博客大纲编写 (Blog Post Outline)
* **任务**：编写关于环保生活技巧的博客大纲。
* **结构化提示词**：
    * **角色 (Role)**：作为一名资深的环保领域内容创作者。
    * **强调 (Emphasis)**：重点介绍日常生活中简单易行的小改动。
    * **限制 (Limitation)**：大纲不宜超过 5 个主要章节。
    * **信息 (Information)**：目标受众是希望减少碳足迹的城市居民。
    * **挑战 (Challenge)**：创建一个既具有教育意义又富有激励性的博客结构。

### 示例 2：新产品市场调研 (Market Research for New Product)
* **任务**：为一款即将发布的新产品进行市场趋势调研。
* **结构化提示词**：
    * **角色 (Role)**：作为一名市场分析专家。
    * **强调 (Emphasis)**：重点关注过去 12 个月内兴起的消费者趋势。
    * **限制 (Limitation)**：仅列出最具潜力的 3 个细分市场。
    * **信息 (Information)**：该产品是一款专为远程办公设计的生产力软件。
    * **挑战 (Challenge)**：分析目标受众的痛点，并找出产品的差异化竞争优势。
