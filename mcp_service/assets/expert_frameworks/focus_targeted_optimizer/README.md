# FOCUS 框架 (FOCUS Framework)

FOCUS 框架通过五个维度的精准定义，帮助用户构建具有高度针对性和实用性的提示词。它特别强调产出物的“实用价值”以及任务的“具体规格”。

## 核心组成部分 (Core Components)

1. **功能 (Function)**：明确定义提示词的功能或目的，概述你希望 AI 做什么。
2. **目标 (Objective)**：指定回答应达到的目标，为 AI 的输出提供一个清晰的靶心。
3. **情境 (Context)**：提供背景信息或特定条件，以丰富 AI 对提示词的理解。
4. **效用 (Utility)**：强调预期响应的有效性或用途，解释这些信息或输出将如何被使用。
5. **规格 (Specifications)**：详细说明对响应的具体要求，如格式、长度或必须解决的关键点。

## 提示词示例 (Prompt Examples)

### 示例 1：业务策略文档 (Business Strategy Document)
* **功能 (Function)**：为一家初创公司起草年度营销策略大纲。
* **目标 (Objective)**：在预算有限的情况下，最大化社交媒体曝光率。
* **情境 (Context)**：公司处于种子轮阶段，专注于有机健康零食。
* **效用 (Utility)**：该大纲将提交给董事会进行审批，作为明年的执行基准。
* **规格 (Specifications)**：以 Markdown 格式分章节呈现，包含 5 个关键动作计划。

### 示例 2：技术支持回复 (Technical Support Response)
* **功能 (Function)**：回复一封关于软件安装失败的客户邮件。
* **目标 (Objective)**：指导用户成功解决安装问题，并降低用户的挫败感。
* **情境 (Context)**：用户正在 Windows 11 上安装 3.0 版本，报错代码为 ERR_005。
* **效用 (Utility)**：提供一个直接可用的回复模板，节省支持团队的处理时间。
* **规格 (Specifications)**：语气要极具同理心，步骤要分点列出（1, 2, 3...），不超过 300 字。
