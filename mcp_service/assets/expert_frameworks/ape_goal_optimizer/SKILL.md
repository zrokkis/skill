# APE 目标驱动优化技能 (APE Goal Optimizer Skill)

本技能通过锁定“目的”与“预期”，消除 AI 在执行任务时的盲目性，确保每一行输出都具备明确的业务价值。

## 执行流程 (Optimization Workflow)

在需要快速下达明确任务指令时，请应用以下 APE 逻辑：

### 1. 动作 (Action) - 行为定义
- **核心动作**：告知 AI 必须完成的具体行为。
- **操作指南**：使用动词，如“提取”、“翻译”、“草拟”、“调试”。

### 2. 目的 (Purpose) - 价值对齐
- **核心动作**：解释这个任务“解决什么问题”或“谁会受益”。
- **操作指南**：描述目标受众的需求或更高层级的项目目标。

### 3. 预期 (Expectation) - 规格把控
- **核心动作**：定义产出物的物理指标和质量标准。
- **操作指南**：指定具体的长度（如“一句话”）、格式（如“JSON”）、语气（如“幽默”）或结构。

## 优化示例 (Standard Template)

**初始需求**：“写个代码审核报告。”

**APE 优化后**：
- **[A] Action**：审查提交的 `OrderService.java` 文件中的代码变更。
- **[P] Purpose**：这是为了在下午的 Code Review 会议前找出潜在的死锁风险和性能瓶颈。
- **[E] Expectation**：
    1. 输出一份 Markdown 格式的问题清单。
    2. 每一项需指出行号、风险等级（高/中/低）及修改建议。
    3. 语言要客观简洁。
