# ELI5 极致简化优化技能 (ELI5 Simplification Optimizer Skill)

本技能通过“比喻驱动”与“术语过滤”，打破知识的傲慢，让信息在不同背景的人群间无缝流动。

## 执行流程 (Optimization Workflow)

在需要向下解析复杂概念或向非专业人士汇报时，请应用以下 ELI5 逻辑：

### 1. 核心解构 - 寻找主干
- **核心动作**：剥离所有技术细节，找出该概念在物理世界中的“第一性原理”。
- **操作指南**：问自己，“如果不谈具体的公式/代码/流程，这件事本质上是在交换什么、改变什么？”

### 2. 类比匹配 - 生活镜像
- **核心动作**：寻找一个 5 岁孩子也玩过的东西或经历过的场景进行对比。
- **操作指南**：使用积木、游乐场、排队、零花钱、玩具交换等生活意象。

### 3. 术语清零 - 语言降噪
- **核心动作**：扫描并替换所有三音节以上的专业词汇。
- **操作指南**：
    - 将“分布式架构”改为“很多个小超市分部在不同地方”。
    - 将“异步机制”改为“你先提交订单，然后去玩，等好了我叫你”。

### 4. 逻辑闭环 - 效果确认
- **核心动作**：确保简化后的逻辑依然是自洽的，且没有产生严重的歧义。

## 优化示例 (Standard Template)

**初始需求**：“跟我解释一下什么是‘机器学习中的梯度下降’。”

**ELI5 优化后**：
- **场景**：你在山顶，但大雾弥漫，你看不到山下的路。
- **比喻**：你的目标是走到山脚下的最低点。
- **动作**：你用脚试探周围的地面，感受哪一个方向的坡度最陡，你就朝那个方向迈出一小步。
- **逻辑**：每走一步，你都要停下来再次试探坡度。只要你一直朝着最斜向下的方向走，最终你一定能走到山底。这个“找最斜坡度并往下走”的过程，就是梯度下降。
