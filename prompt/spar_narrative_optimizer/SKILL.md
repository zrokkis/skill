# SPAR 深度任务导向优化技能 (SPAR Task-Oriented Skill)

本技能通过锁定任务的物理基础（场景）与最终价值（结果），强制 AI 在执行动作时必须时刻关注“为什么要这么做”以及“最终要交付什么”，从而彻底解决输出内容虽然正确但由于缺乏背景而无法使用的通病。

## 执行流程 (Optimization Workflow)

在需要请求 AI 协助处理具有明确背景和目标的任务时，请应用以下 SPAR 逻辑：

### 1. 场景 (Situation) - 绘制基座
- **核心动作**：交代“此时此刻”发生了什么。
- **操作指南**：描述当前的瓶颈、现有的资源或面临的紧急情况，让 AI 能够“共情”你的处境。

### 2. 目的 (Purpose) - 价值锚点
- **核心动作**：声明这次互动的核心战略意图。
- **操作指南**：不要只是说“写一段话”，要说“为了通过某项审批”、“为了说服某位客户”或“为了弄清某个逻辑漏洞”。

### 3. 行动 (Action) - 精准指派
- **核心动作**：下达具体的指令。
- **操作指南**：使用强动词，如“提炼”、“重构”、“压力测试”、“模拟”等。

### 4. 结果 (Result) - 定义成功
- **核心动作**：给出一个可感知的、具体的交付标准。
- **操作指南**：描述成功后的状态，例如“一份无须修改即可发送的草稿”、“一套覆盖全场景的测试用例”。

## 优化示例 (Standard Template)

**初始需求**：“帮我写个项目进展报告。”

**SPAR 优化后**：
- **[S] Situation**：我们正在进行的“自动化客服升级项目”目前进度已过半，但在与第三方 API 对接时遇到了意料之外的技术延迟。
- **[P] Purpose**：我需要向项目指导委员会汇报这一情况，目标是让他们理解问题的严重性，并申请额外的技术专家支持。
- **[A] Action**：请根据上述描述撰写一份周报，突出我们已完成的里程碑，并以客观但紧迫的笔触解释目前的技术瓶颈。
- **[R] Result**：输出一个 300 字左右的周报段落，要求包含“进展”、“风险”和“待办事项”三个小标题。
