# CIDI 结构化决策优化技能 (CIDI Prompt Optimizer Skill)

本技能通过强化的“数据输入”与“细节约束”，确保 AI 的产出不仅准确，而且具备极高的可落地性。

## 执行流程 (Optimization Workflow)

在处理需要严谨结构和数据支撑的任务时，请应用以下 CIDI 逻辑：

### 1. 背景 (Context) - 场域构建
- **核心动作**：定义角色并划定权责边界。
- **操作指南**：明确 AI 是作为“审计员”、“开发者”还是“创意总监”。告知它在哪个“宇宙”里工作。

### 2. 指令 (Instructions) - 行为描述
- **核心动作**：分步骤下达命令。
- **操作指南**：使用清晰的序列，如“步骤 1：分析 X；步骤 2：提取 Y；步骤 3：生成汇报 Z”。

### 3. 细节 (Details) - 精度控制
- **核心动作**：添加参数化限制。
- **操作指南**：指定特定的格式（JSON, Table）、语气（正式、幽默）、长度限制以及“必须避开”的禁忌点。

### 4. 输入 (Input) - 实况喂料
- **核心动作**：提供原材料。
- **操作指南**：粘贴你需要处理的原始文本、代码片段、市场调研数据或用户反馈记录。

## 优化示例 (Standard Template)

**初始需求**：“帮我整理一下这些用户评价。”

**CIDI 优化后**：
- **[C] Context**：你是一位用户体验（UX）研究员。
- **[I] Instructions**：分析输入的评价，将其分类为“功能建议”、“Bug 报告”和“情感反馈”。
- **[D] Details**：以 Markdown 表格输出。每一行必须包含：分类、评价原文摘录、优先级（P0/P1/P2）。
- **[I] Input**：[在此粘贴 50 条原始用户评价...]
