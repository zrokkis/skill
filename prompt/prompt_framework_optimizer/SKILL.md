# PROMPT 全维度构建技能 (PROMPT Comprehensive Skill)

本技能通过六个维度的精细调控，如同调音台一样对 AI 的输出进行全方位的“参数设置”。它不仅关注内容本身，还严格控制内容的呈现形式、风格基调和边界条件，是追求完美输出的必备工具。

## 执行流程 (Optimization Workflow)

在需要对生成内容进行精细化控制时，请应用以下 PROMPT 逻辑：

### 1. 角色 (Persona) - 灵魂注入
- **核心动作**：定义 AI 的职业背景和思维模式。
- **操作指南**：使用“你是一位拥有 10 年经验的 [职位]”或“以 [著名人物] 的风格思考”。

### 2. 请求 (Request) - 核心指令
- **核心动作**：明确具体的任务动作。
- **操作指南**：使用强动词，如“撰写”、“调试”、“翻译”、“总结”。

### 3. 输出类型 (Output Type) - 容器定义
- **核心动作**：指定交付物的物理形态。
- **操作指南**：明确是“HTML 表格”、“JSON 数据”、“Markdown 文档”还是“Python 脚本”。

### 4. 修饰符 (Modifier) - 边界约束
- **核心动作**：设定限制条件和优化方向。
- **操作指南**：使用“不要使用被动语态”、“字数限制在 500 字以内”、“侧重于成本效益分析”。

### 5. 提供示例 (Provide Example) - 锚点校准
- **核心动作**：展示一个标准的“少样本 (Few-Shot)”。
- **操作指南**：提供一段理想的输出范例，或者指定参考某个已知的标准（如“参考 RFC 规范格式”）。

### 6. 语气 (Tone) - 情感调色
- **核心动作**：设定文字的温度和色彩。
- **操作指南**：指定“正式”、“幽默”、“同理心强”或“像海明威一样简练”。

## 优化示例 (Standard Template)

**初始需求**：“给客户写封道歉信。”

**PROMPT 优化后**：
- **[P] Persona**：你是一位资深的客户成功经理，擅长危机公关。
- **[R] Request**：撰写一封致核心客户的道歉信，解释上周服务中断的原因。
- **[O] Output Type**：一封格式标准的商务电子邮件。
- **[M] Modifier**：
    1. 必须提及我们已采取的 3 项预防复发的措施。
    2. 避免推卸责任给第三方供应商。
- **[P] Provide Example**：开头请参考：“尊敬的 [客户名]，如果是我的服务出现了这样的问题，我也会感到同样的沮丧...”
- **[T] Tone**：诚恳、专业、负责任，且不卑不亢。
