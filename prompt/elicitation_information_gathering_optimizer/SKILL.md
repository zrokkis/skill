# 引导式信息采集优化技能 (Elicitation Info Gathering Optimizer Skill)

本技能通过锁定“意图”与“跟进策略”，将 AI 的交互模式从被动回答转变为主动、深度的知识挖掘，确保获取的信息具备深度和高度的相关性。

## 执行流程 (Optimization Workflow)

在需要进行市场调研、需求调研或复杂问题拆解时，请应用以下引导逻辑：

### 1. 明确意图 (Clarify Intent) - 动机声明
- **核心动作**：告知 AI 你为什么需要这些信息。
- **操作指南**：使用“我正在进行 [项目名称] 的前期调研，目的是为了 [具体的商业/学术目标]”。

### 2. 结构化问题 (Structure Questions) - 开放式挖掘
- **核心动作**：抛弃封闭式问题，使用引导思考的提问方式。
- **操作指南**：使用“请详细描述...”、“探讨...的可能性”、“分析...背后的机制”等句式。

### 3. 提供上下文 (Contextualize) - 背景灌注
- **核心动作**：为问题设置边界和物理环境。
- **操作指南**：描述行业背景、目标市场、技术现状或特定的用户群体。

### 4. 指定预期结果 (Specify Desired Outcome) - 格式锚定
- **核心动作**：规定产出的形态。
- **操作指南**：指定是“SWOT 分析表”、“叙事性综述”还是“分步骤的执行路线图”。

### 5. 战略性跟进 (Follow-Up Strategically) - 迭代下钻
- **核心动作**：预告下一步动作，保持交互的连贯性。
- **操作指南**：告诉 AI “在完成上述分析后，请准备好针对 [特定细节 X] 进行更深入的探讨”。

## 优化示例 (Standard Template)

**初始需求**：“告诉我一些关于 Web3 游戏的事情。”

**引导式优化后**：
- **[Intent]**：我正在准备一份关于“Web3 游戏如何提升玩家留存率”的内部分析报告。
- **[Question]**：请深入探讨 Play-to-Earn 模式与玩家长期忠诚度之间的关系。
- **[Context]**：考虑到近期加密市场的波动以及玩家对游戏性要求的提升。
- **[Outcome]**：请列出 3 个成功案例和 3 个失败教训，以 Markdown 表格对比呈现。
- **[Follow-Up]**：在提供以上分析后，我将要求你为我们即将开发的新游戏设计一套 Token 激励模型。
