# SPEAR 提示词迭代优化技能 (SPEAR Prompt Optimizer Skill)

本技能强调“人类直觉”与“循环改进”，将提示词编写过程简化为一种自然的对话与反馈机制。

## 执行流程 (Optimization Workflow)

在需要快速解决问题并希望与 AI 进行深度协作时，请应用以下 SPEAR 逻辑：

### 1. 开始 (Start) - 目标声明
- **核心动作**：直接点出任务主题。
- **操作指南**：使用动词起始，如“计划一个...”、“起草一份...”。

### 2. 提供 (Provide) - 样式喂料
- **核心动作**：给样板，定格式。
- **操作指南**：提供一个参考示例或明确的输出模板。

### 3. 解释 (Explain) - 语境说明
- **核心动作**：告知背景。
- **操作指南**：假装你在和新入职的同事说话，把项目的来龙去脉、特定偏好和当前进度说清楚。

### 4. 要求 (Ask) - 核心指令
- **核心动作**：临门一脚，明确真正的诉求。
- **操作指南**：用具体的疑问词或祈使句，如“请生成 X”、“你认为 Y 的风险在哪？”。

### 5. 润色与重复 (Rinse & Repeat) - 持续进化
- **核心动作**：根据初稿反馈，再次执行以上步骤。
- **操作指南**：告诉 AI “这一版不错，但请把语气调得再正式一点”或“请补充关于成本的分析”。

## 优化示例 (Standard Template)

**初始需求**：“写个旅行计划。”

**SPEAR 优化后**：
- **[S] Start**：我想去北京玩 3 天。
- **[P] Provide**：请按“上午/下午/晚上”的时间表来排，加上大约的预算。
- **[E] Explain**：我是和年迈的父母一起去，所以行程不能太赶，不需要极限打卡，多一点文化体验和舒适的餐饮。
- **[A] Ask**：请根据这些需求生成一个轻松的北京文化之旅攻略。
- **[R] Rinse**：如果景点门票需要预约，请在最后提醒我。
