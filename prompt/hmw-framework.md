# HMW 框架 (How Might We Framework)

HMW 框架（我们如何能够）是一种源自设计思维（Design Thinking）的创新工具。它通过将挑战重构为以“我们如何能够...”开头的开放式提问，激发团队的协作精神，打破思维定式，从而发现更具影响力和创新性的解决方案。

## 核心组成部分 (Core Components)

1. **开放式提问 (Open-Ended Inquiry)**：将面临的挑战或痛点重构为一个开放式问题，以鼓励更广泛的解决方案组合。
2. **促进协作 (Promote Collaboration)**：旨在促进团队集体构思，鼓励所有成员贡献想法，利用多元化的视角。
3. **鼓励创新思维 (Encourage Innovative Thinking)**：通过消除对解决方案类型的限制，推动参与者跳出传统方案的窠臼。
4. **促进方案发现 (Facilitate Solution Discovery)**：引导团队进入发现过程，将焦点放在寻找创新方案上，而非纠结于问题本身。

## 提示词示例 (Prompt Examples)

### 示例 1：数字化参与度提升 (Digital Engagement Improvement)
* **背景**：一家公司寻求提高其数字化平台的客户参与度。
* **结构化提示词 (HMW 问题)**：
    * **开放式提问 (Open-Ended Inquiry)**：我们如何能够重新设计我们的数字化平台，使其对用户更具吸引力？
    * **促进协作 (Promote Collaboration)**：我们如何能够从用户那里收集实时反馈，以指导我们的设计过程？
    * **鼓励创新思维 (Encourage Innovative Thinking)**：我们如何能够集成社交功能，以在用户之间建立社区感？
    * **促进方案发现 (Facilitate Solution Discovery)**：我们如何能够利用游戏化元素来激励用户每天登录？

### 示例 2：优化客户支持 (Optimizing Customer Support)
* **背景**：客户支持团队面临响应时间过长的问题。
* **结构化提示词 (HMW 问题)**：
    * **开放式提问 (Open-Ended Inquiry)**：我们如何能够通过自动化工具缩短首响时间，同时不降低服务的人性化程度？
    * **创新思维 (Innovative Thinking)**：我们如何能够让客户在等待期间也能感觉问题正在被处理，甚至自行解决部分问题？
