# CRISPE 提示词优化技能 (CRISPE Prompt Optimization Skill)

本技能通过 CRISPE 框架的六个维度，强制 AI 在预设的“边界”内进行高质量创作。

## 执行流程 (Optimization Workflow)

在处理高精密任务时，请应用以下 CRISPE 逻辑：

### 1. 清晰度 (Clarity) - 任务定义
- **核心动作**：用一句话说清你要 AI 做什么。
- **操作指南**：使用强动词。避免“帮我写点东西”，改用“撰写一份技术规格书”。

### 2. 相关性 (Relevance) - 背景对齐
- **核心动作**：告知 AI 为什么要做这个，以及给谁看。
- **操作指南**：明确受众（如：初学者 vs 专家）和应用场景。

### 3. 迭代 (Iteration) - 逐步求精
- **核心动作**：不要指望一次完美，建立反馈心智。
- **操作指南**：告知 AI “如果输出不符合预期，我会提供反馈进行第二轮修改”。

### 4. 针对性 (Specificity) - 细节约束
- **核心动作**：提供具体的关键词、数据点或必须包含的内容。
- **操作指南**：明确“必须包含 [X]”或“绝对不要提及 [Y]”。

### 5. 参数 (Parameters) - 格式控制
- **核心动作**：定义输出的物理属性。
- **操作指南**：
    - **字数**：不超过 500 字。
    - **格式**：JSON、Markdown 表格、多级大纲。
    - **语气**：正式、冷峻、同理心风。

### 6. 示例 (Examples) - 模式锚定
- **核心动作**：给出参考模板。
- **操作指南**：至少提供一个正确的输出范本。

## 优化示例 (Standard Template)

**初始需求**：“写个代码审查列表。”

**CRISPE 优化后**：
- **[C] 清晰度**：制定一份针对 Python 后端代码的自动化审查清单。
- **[R] 相关性**：受众是初级开发者，目的是统一团队的代码规范，减少运行时报错。
- **[I] 迭代**：请先生成初稿，我会根据团队特定库的使用情况提出修改意见。
- **[S] 针对性**：必须涵盖异常处理、日志记录规范和内存泄露检查。
- **[P] 参数**：以 Markdown Checkbox 列表形式输出，按优先级（高/中/低）分类。
- **[E] 示例**：
    - [ ] 是否在所有外部调用处使用了 try-except 块？
    - [ ] 日志是否包含 Traceback？
