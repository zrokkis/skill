# ORID 聚焦会谈法 (ORID Focused Conversation Method)

ORID 框架是一种由文化事业协会 (ICA) 开发的结构化会谈方法。它通过四个连续的提问层面，引导个人或群体从客观事实出发，经历情感反应和深度诠释，最终做出明智的决定。在提示词工程中，它能帮助 AI 进行深度的案例复盘、事件分析或决策制定。

## 核心组成部分 (Core Components)

1. **客观层面 (Objective)**：关注事实、数据和感官信息。这是关于“我看到了什么？”、“发生了什么？”的层面。
2. **反应层面 (Reflective)**：挖掘情感响应、心情、感觉或直觉。这是关于“我的初衷是什么？”、“我有什么感受？”的层面。
3. **诠释层面 (Interpretive)**：探索意义、价值和重要性。这是关于“这意味着什么？”、“我们学到了什么？”的“所以呢？”层面。
4. **决定层面 (Decisional)**：确定未来的决议、行动或决定。这是关于“我们下一步要做什么？”的“现在该怎么办？”层面。

## 提示词示例 (Prompt Examples)

### 示例 1：项目失败复盘 (Project Failure Review)
* **任务**：对一个未能按时交付的软件项目进行复盘。
* **结构化提示词**：
    * **客观 (Objective)**：列出项目从启动到截止日期的关键里程碑以及实际完成情况。
    * **反应 (Reflective)**：团队在面对进度压力时的整体士气如何？哪些环节让你感到最沮丧？
    * **诠释 (Interpretive)**：既然进度滞后了，这反映了我们在资源评估或沟通流程上存在哪些深层问题？
    * **决定 (Decisional)**：针对下个项目，我们需要实施哪 3 项具体的流程改进以避免类似情况发生？

### 示例 2：客户访谈分析 (Customer Interview Analysis)
* **任务**：分析一段用户测试视频。
* **结构化提示词**：
    * **客观 (Objective)**：记录用户在点击“下单”按钮前的具体动作和犹豫时间。
    * **反应 (Reflective)**：用户在操作失败时表现出了什么样的情绪（如困惑、愤怒或沉默）？
    * **诠释 (Interpretive)**：这种困惑是否意味着我们的 UI 布局逻辑违背了大众的直觉？
    * **决定 (Decisional)**：请给出该页面的重构建议，并列出最高优先级的修改项。
