# 3Cs 提示词质量优化技能 (3Cs Quality Optimizer Skill)

本技能通过三个维度对现有提示词进行“体检”，消除模糊表达，强化逻辑连接，锁定输出风格。

## 执行流程 (Optimization Workflow)

在处理需要高度专业性或风格敏感的任务时，请应用以下 3Cs 逻辑进行自我审查：

### 1. 清晰度 (Clarity) - 消除迷雾
- **核心动作**：检查是否有歧义词汇。
- **操作指南**：
    - 问“AI 有可能理解成别的意思吗？”
    - 如果任务是“写个关于猫的文章”，改为“撰写一篇关于布偶猫喂养注意事项的科普短文”。

### 2. 连贯性 (Coherence) - 逻辑链条
- **核心动作**：检查指令之间的顺序和衔接。
- **操作指南**：
    - 确保先定义前提，再下达动作，最后规定格式。
    - 指令流应像剥洋葱一样，层层递进。

### 3. 一致性 (Consistency) - 风格定型
- **核心动作**：检查语调、语气和术语的使用是否统一。
- **操作指南**：
    - 如果开头要求“专业严谨”，结尾就不应出现“开个玩笑”。
    - 确保所有关键术语（如：转化率 vs 转换率）在全文中保持一致。

## 优化示例 (Standard Template)

**初始需求**：“写个关于团队合作的代码注释，要好一点，前后都要有逻辑。”

**3Cs 优化后**：
- **[Clarity]**：请为提供的 Java 函数生成详细的 JavaDoc 注释。
- **[Coherence]**：
    1. 首先简述函数的功能目标。
    2. 接着列出所有输入参数的含义。
    3. 最后说明返回值及其在异常情况下的行为。
- **[Consistency]**：全篇使用阿里巴巴 Java 编程手册规定的学术语调，不要包含非正式的幽默语。
